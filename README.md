Online Bus Reservation System – System Modeling and Agile Development

This project is about designing an online bus reservation system. The main aim was to understand how a software system is planned before actual development begins. Instead of focusing on coding, more attention was given to system modeling and organizing the development process.

The system allows users to search for buses between locations, select seats, book tickets, make payments, and cancel bookings. While working on this, it became clear that even a simple system has multiple steps and interactions that need to be planned properly.

To represent the system, different UML diagrams were created. A context diagram shows how the system interacts with external entities such as users, payment gateway, and bus operators. A sequence diagram explains the step-by-step process of booking a ticket. A state diagram shows how the system moves through different stages like searching, selecting, payment, and booking.

Agile methodology was also applied using a simple Scrum approach. The work was divided into smaller parts, and user stories were written to define what needs to be done. This helped in managing the work better and keeping things clear.

The tools used for this project include Draw.io for creating diagrams, GitHub for storing files, and Microsoft Word for documentation. The repository contains UML diagrams and the project report.

This is a design-based academic project created for learning purposes. No actual implementation or coding has been done.
